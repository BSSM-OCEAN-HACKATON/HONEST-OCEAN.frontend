# 커밋 컨벤션

## 커밋 메시지 형식

커밋 메시지는 항상 다음 형식을 따라야 합니다:

```
<type> : <content>
```

### 타입 (Type)

다음 타입들을 사용할 수 있습니다:
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `refactor`: 코드 리팩토링
- `chore`: 빌드 업무 수정, 패키지 매니저 설정 등
- `style`: 코드 포맷팅, 세미콜론 누락 등 (코드 변경이 없는 경우)
- `docs`: 문서 수정
- `test`: 테스트 코드 추가 또는 수정
- `perf`: 성능 개선

### 내용 (Content)

- 내용은 **항상 한국어로 작성**해야 합니다.
- 기능별, 컨텍스트별로 변경사항을 분리하여 커밋합니다.
- 간결하고 명확하게 작성합니다.

### 예시

```
feat : 회원가입 API 오류 수정
refactor : 컴포넌트 구조 개선
chore : 의존성 패키지 업데이트
fix : 카메라 스캔 페이지 렌더링 오류 수정
```
